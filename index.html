Research Interests

I am interested in interpretability as a concrete path toward making advanced AI go well. The four directions below map the problems I am currently thinking about and building toward.

  • Reasoning-model interpretability — Causal analysis of chain-of-thought: which reasoning steps actually drive a model's answer, identified via counterfactual resampling and intervention.
  • Internals-based monitoring — Cheap linear probes trained on internal activations to detect behaviors like deception and evaluation-awareness, as a first layer of defense in model control schemes.
  • Automating interpretability — Autonomous "investigator agents" that use interpretability tools to explain why a model behaved a certain way, closing the loop between tool-using agents and mechanistic analysis.
  • Latent-reasoning & diffusion-LLM interpretability — Keeping models interpretable as the field moves toward vector-based and diffusion-based reasoning that breaks chain-of-thought monitoring.

Selected Projects

  • Diffusion Detective: Interpretability Tooling for a Generative Model
    A research toolkit for peering inside a generative AI model as it draws: it shows what the model "focuses on" at each step, lets you nudge its behavior by adding a concept vector mid-generation (94% steering success rate), and writes a plain-English explanation of what changed and why. The same three operations - extract internals, intervene causally, explain in natural language are the core primitives of LLM mechanistic interpretability (activation steering, patchscopes).
    GitHub
  • Affect-Diff: Causal-Diffusion Bridge for Interpretable Emotion Recognition
    An AI model that doesn't just predict emotion, it tells you which part of a person's communication (words, tone, or face) causally drove that prediction. A NOTEARS-learned causal graph re-weights modalities, a β-VAE compresses them to a shared latent space, and a stop-gradiented 1D DDPM prior structures it for generation; counterfactual sampling then makes attributions explicit. Result: 18% relative improvement over the strongest baseline on CMU-MOSEI, with non-trivial performance on minority emotions where all baselines fail entirely.
    arXiv Paper page GitHub
  • Causal Factor Discovery in S&P 500 Returns
    Causal-inference toolkit: Double Machine Learning (DML) for ATE/CATE estimation of momentum on next-day returns. The same counterfactual machinery now central to reasoning-model interpretability, orthogonalizing out confounders to isolate which factors causally drive an outcome.
    GitHub
  • No-Propagation Diffusion Transformers
    From-scratch reproduction and extension of a recent Oxford paper (NoPropDT, arXiv:2503.24322). Demonstrates the empirical reproduction skill that interpretability research runs on rebuilding an architecture layer by layer to understand it mechanically. (~99% MNIST, ~76% CIFAR-10 without backpropagation.)
    GitHub
  • Autonomous Tool-Using Systems — the same read-reason-act pattern underlying autonomous interpretability investigator agents
  • MAESTRO: AI Text-to-Video Lecture System
    Autonomous pipeline (OpenAI script synthesis + PlayHT TTS + FFmpeg rendering) that generates educational video lectures from text and answers student questions via RAG. Demonstrates end-to-end multi-agent orchestration, same architecture pattern underlying autonomous interpretability investigators. Live at Fordham University.
    GitHub Live
  • AutomatedBI: Agentic BI Dashboard Generator
    CrewAI multi-agent system (4 specialized agents) + Gemini: reads raw data, reasons with domain-expert and KPI-strategist agents, and produces an interactive dashboard. An instance of the general read-input / reason-with-tools / produce-result pattern that underlies autonomous interpretability agents. 80% reduction in manual reporting time.
    GitHub Demo
  • H-KFX: Autonomous Bug-Fixing Agent
    GPT-4o + MCP + MongoDB Atlas RAG agent that closes Jira tickets end-to-end: reads the ticket, retrieves context, generates and validates a fix, and opens the PR , 60–90 second ticket-to-PR cycle. A working instance of the autonomous investigator architecture: the same scaffold needed for agents that probe why a model behaved a certain way. MUG Hackathon 2025.
    GitHub Demo
  • RamsNavigator: AI Campus Navigation
    Hybrid Neo4j (Dijkstra routing) + MongoDB + GPT-4o-mini natural language parsing for indoor campus navigation. React + Cytoscape.js interactive graph map.
    GitHub
